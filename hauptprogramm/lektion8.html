Lektion 8: Daten sortieren und begrenzen – ORDER BY & LIMIT

ORDER BY – Ergebnisse sortieren

Mit ORDER BY kannst du Daten auf- oder absteigend sortieren. Die Sortierung erfolgt standardmäßig aufsteigend (ASC), kann aber mit DESC auch absteigend erfolgen.

SELECT * FROM Artikel ORDER BY Preis ASC;
SELECT * FROM Artikel ORDER BY Preis DESC;

Auch eine mehrfache Sortierung ist möglich:

SELECT * FROM Kunde ORDER BY Ort ASC, Name DESC;

LIMIT – Ergebnismenge begrenzen

Mit LIMIT kannst du festlegen, wie viele Zeilen zurückgegeben werden sollen. Besonders praktisch in Verbindung mit ORDER BY, z. B. um nur die teuersten 3 Artikel zu sehen:

SELECT * FROM Artikel ORDER BY Preis DESC LIMIT 3;

Beispiele für Sortierkombinationen

Sortierregel SQL-Beispiel
Nur nach einer Spalte (aufsteigend) ORDER BY Preis ASC
Nur nach einer Spalte (absteigend) ORDER BY Preis DESC
Nach zwei Spalten, beide aufsteigend ORDER BY Ort ASC, Name ASC
Nach Ort (aufsteigend), dann Name (absteigend) ORDER BY Ort ASC, Name DESC
Nach Datum, dann Preis ORDER BY Bestelldatum DESC, Preis ASC

Wichtig zu beachten

  • ORDER BY kommt immer nach WHERE
  • LIMIT steht am Ende der Abfrage

Aufgabe 1

Gib alle Artikel sortiert nach Preis aufsteigend aus.

Aufgabe 2

Zeige die drei teuersten Artikel (absteigend sortiert).

Aufgabe 3

Sortiere alle Kunden zuerst nach Ort (aufsteigend) und dann nach Name (absteigend).
